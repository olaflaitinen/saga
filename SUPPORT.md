# Support

## How to get help

1. **Read the documentation** at the GitHub Pages site (built from `docs/`) or browse
   the `docs/` directory directly. The [FAQ](docs/faq.md) covers the most common questions.

2. **Search existing issues** at https://github.com/olaflaitinen/saga/issues before
   opening a new one.

3. **Open a GitHub Issue** using the appropriate template:
   - [Bug report](https://github.com/olaflaitinen/saga/issues/new?template=bug_report.yaml)
   - [Feature request](https://github.com/olaflaitinen/saga/issues/new?template=feature_request.yaml)
   - [Reproducibility question](https://github.com/olaflaitinen/saga/issues/new?template=reproducibility_question.yaml)

4. **Email the corresponding author** at olaf.laitinen@su.se for questions related to
   data access, ethics approval, or MONA project access that cannot be discussed publicly.

## Response times

This repository is maintained on a best-effort basis alongside active academic research.
Typical response time for GitHub Issues is 5 to 14 business days. For critical bugs
(model output errors, security vulnerabilities), responses will be prioritized.

## Out-of-scope support

The authors cannot provide:
- Technical support for hardware or CUDA driver issues unrelated to SAGA.
- Support for uses that violate the dual-use statement (see [docs/ethics/dual-use-statement.md](docs/ethics/dual-use-statement.md)).
- Access to the real LISA microdata (requests for MONA project access must go through Statistics Sweden directly).
